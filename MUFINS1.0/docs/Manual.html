JyMet user manual

JyMet (version 2.34) is a graphical user interface to the program SurreyFBA, which is a command-line tool for metabolic model analysis. JyMet enables creating, editing, saving and loading metabolic models, definition of modelling problems in a simple scripting format and their solution using SurreyFBA, as well as visualisation of models and results. This manual is about JyMet only, more details about SurreyFBA can be found in its manual (further referred as SFM).

 

1  Model file structure

A JyMet model consists of four hierarchical levels: genes, enzymes, reactions and metabolites. An element of each level can be associated with elements on the higher level, called parents (e.g. genes encoding an enzyme, enzymes catalysing a reaction, or reactions using (i.e. consuming or producing) a metabolite, and on the lower level, called children.

Models can be represented in one of the following tab-delimited formats:

  • SBML format: a widely used model representation format based on XML, which defines model into classified components, currently only Level 2 Version 1 being supported.
  • `simple' format: which describes only reactions and is also accepted by SurreyFBA (see SFM). File can be in text format or Excel ‘xls’ workbook format.
  • `extended' format: which consists of four parts, describing the hierarchical levels of the model. File can be in text format or Excel ‘xls’ workbook format.

Each part starts with the name of the corresponding hierarchical level, followed by lines describing individual elements. The description of each element consists of tab-delimited values of its attributes, listed below:

  • Genes: ID, Name, Children, Predictions, Comment.
  • Enzymes: ID, Name, Children, Rule, Compartments, Comment.
  • Reactions: ID, Name, Equation, LB, UB, Rule, Comment.
  • Metabolites: ID, Name, External, Canonical SMILES, Isomeric SMILES, Molecular mass, Comment.

The meaning of the attributes is shown below:

  • ID: a modelwide unique identifier
  • Name: arbitrary string, may be non-unique
  • Children: list of the children’s IDs.
  • Predictions: list of BLAST predictions. Each prediction consists of an EC number followed by a colon and the e-value.
  • Rule: a logical rule representing the association with the parents; may include IDs, AND and OR operators (case insensitive), and parentheses, all space-delimited. E.g. the following rule: `g1 and ( g2 or g3 )' means that the given enzyme is encoded by the genes g1 and either g2 or g3.
  • Equation: a reaction equation, consisting of metabolite identifiers, stoichiometric coefficients (the coefficient 1 can be skipped), `+' and `=' operators, all space-delimited.
  • LB: a floating point number defining the lower flux rate bound.
  • UB: a floating point number defining the upper flux rate bound.
  • External: Boolean value `True' if the metabolite is external and `False', otherwise. If you set the externals by specifying a tag at Solve > Externality tag, the externals specified by Boolean value or problem table will be overwritten.
  • Compartment, Canonical SMILES, Isomeric SMILES, Molecular mass: arbitrary strings.
  • Comment: arbitrary string starting with the hash symbol.

Note that a reaction equation may be empty (i.e. consist of a `=' operator only) and a metabolite may be used by no reactions.

 

2  Model editor

JyMet can be used for work with multiple models simultaneously. Each model is shown in a separate tab, whose title shows its name (if it is newly created) or the complete path (if it is loaded from a file). A model tab consists of five lower-level tabs, titled `Genes', `Enzymes', 'Reactions', `Metabolites' and `Problem'. Each tab contains an empty table (spreadsheet); the first four describe the corresponding hierarchical levels of the model, with rows corresponding to the elements and columns corresponding to the attributes, as described in Section 1. Note that the `Metabolites' table contains an additional column `Used by', which shows the reactions using the given metabolites (it is filled in accordance with the equations). The `Problem' table is described in Section 3.

Creating a model  Select File > New model in the menu and enter a model name into the field in the dialog window. A new model tab with empty tables will appear.

Adding elements   Select the table showing the hierarchical layer to which new elements are to be added; then select Edit > Insert rows and enter the space separated names of the new elements into the appearing dialog box.

Deleting elements   Select the rows (i.e. any cells in those rows) describing the elements to be deleted, then select Edit > Delete rows. Note that a gene or an enzyme cannot be deleted if it has children, so the "Rule" attribute of the corresponding children has to be edited first (see below).

Editing elements   Attributes are edited by simply entering new values into the corresponding cells. Note that the columns `Children' and `Used by' are not editable. If a new value of a `Rule' or `Equation' attribute includes new IDs, new elements with these IDs are included into the model and the corresponding rows appear in tables.

Searching and filtering table contents   Select Edit > Search and enter a regular expression into the appearing dialog box to search for it in the table currently selected. The table will be filtered to the rows containing matches to this expression. To limit the search to a subset of columns, select them before the search.

Select View > Show > Children or View > Show > Parents to filter the adjacent table to the rows describing the children or the parents of the elements currently selected, respectively.

Select View > Show > All to unfilter a table.

Opening a model   Select File > Open model and select a file in one of the formats described in Section 1. A new model tab with tables populated with the model contents will appear.

Saving a model   Select File > Save model and specify a file name. The model will be saved in the `extended' format (see Section 1).

Importing a SBML model  Select File > Import SBML and select a file name (xml file).  The SBML file will be parsed and loaded and model tabs will appear.

Exporting a SBML model  Select File > Export SBML and specify a xml file name. The model will be converted into SMBL model format, where making sure externality tag is specified (in Solve > Externality tag) if the externality tag is not ‘_b’. To conform to the Sid type for SBML components, apart from  characters of letters, numbers and underscores, some special characters are replace with '_'+ Abbreviation + '_', for example: ',' replaced with '_ COMMA_'. Because Model name is file name, the name should be in Sid type.

Saving a table   Select File > Save table and select a file name. The currently selected low-level tab will be saved in a file, whose format depends on the contents of the tab (tab-delimited for tables and graphical files for diagrams). If a `Reactions' table is selected, the resulting file will be in the `simple' model format (see Section 1).

Printing   Select File > Print to print the contents of the tab currently selected.

 

3  Problem solver

JyMet uses SurreyFBA to solve various metabolic modelling problems, launching it as a separate background process. Before a problem or a sequence of problems can be solved, they have to be defined in the `Problem' table associated with the model to which it is to be applied. The content of this table is then passed to SurreyFBA as the problem file. The output of the solution is shown in a separate table and can be visualised in a diagram or a plot.

Defining a problem   Each row in a `Problem' table contains one line of the problem file, whose syntax can be found in SFM. The `Expression' column may contain the following data:

  • ‘!dir: obj_fun' where dir is either "min" for minimisation or "max" for maximisation, and obj_fun is the objective function, defined as a linear expression including reaction or metabolite names, stoichiometric coefficients (the coefficient 1 can be skipped), and `+' operators, all space-delimited, e.g. `m1 + 0.5 r2'. Note that the `-' operators are not allowed, so negative stoichiometric coefficients should be used instead, e.g. `m1 + -0.5 r2'.
  • ‘? arg_list' where arg_list is the argument list in a method-specific format (see below).
  • ‘\ ext_list' where ext_list is a space-delimited list of external metabolites, overriding the list defined in the model.
  • a space-delimited linear expression (the syntax is the same as for objective functions).
  • ‘;' end of record.
  • For (Fast) iMAT method, ‘!expression_name’ indicates which column of gene/enzyme expression profile in the expression table, only one name for a record.
  • For GIM3E method, ‘!objective,array’ specifying an objective reaction and a metabolic array of metabolic signal profile, delimited by a comma.
  • For problems SGNI and WGNI, ‘!nutrient_names’ defines the space of growth media under which we perform prediction of GNIs, names are single space-delimited; and ‘$nutrient names’ designates the nutrients to be always present and rest nutrients will be set absent in the rest nutrients.

The objective function, the argument list, and the list of external metabolites apply to all subsequent records until redefined.

The `LB' and `UB' columns can only contain the lower and upper bounds of the linear expressions defined in the corresponding rows, thus forming linear constraints. If the upper bound is skipped, the value is fixed to the lower bound. The constraints apply everywhere once defined.

The `Comment' column contains comments. Note that comments are forbidden in all rows except those describing linear constraints.

To insert new rows into the `Problem' table, select Edit > Insert rows and enter the number of rows into the appearing dialog box. To delete rows, select them and select Edit > Delete rows.

To save a problem, select the `Problem' table and File > Save table; the problem will be saved in the tab-delimited format of SurreyFBA problem files. To open a problem file, select File > Open problem; the contents will appear in the `Problem' table.

Analysis methods   JyMet implements a range of analysis methods, corresponding to the SurreyFBA `program' options. The list of the methods and their specific argument list formats is shown below:

  • Objective value:compute the optimal objective function value and show it (no args)
  • Objective value and status:compute the optimal objective function value, show it and the solution status, e.g. OPTIMAL, FEASIBLE, UNDEFINED etc. (no args)
  • FBA: (Flux Balance Analysis) compute a flux distribution corresponding to an optimal value of the objective function (no args)
  • FVA: (Flux Variability Analysis) compute the minimal and maximal flux rates of the query reactions, corresponding to the optimal objective function value (args: space-separated list of reactions; if not provided, all reactions are considered)
  • Elementary mode: compute one or more elementary flux mode involving the query reaction. The objective is to minimise the number of reactions involved (if the solver is MILP), or the sum of flux rates in the flux distribution (otherwise). The first argument is a reaction to be involved in the mode. The second argument is the number of modes to be computed (1 by default). If the solver is MILP, an additional scaling factor between 0 and 1 can be provided, since all flux rates in the resulting distribution must be in this range. The default scaling factor is 0.001; note that decreasing it may result in longer solution times, while increasing may result in the impossibility of a solution. (args: `reaction [number, scale_factor]').
  • Knock-out analysis: same as objstat, but the query reactions/enzymes are knocked out (args: space-separated list of reactions/enzymes)
  • Essential reactions: detect the essential reactions, by blocking which the value of the objective function becomes zero (args: space-separated list of query reactions; if not provided, all reactions are considered)
  • Live reactions: detect the live reactions, i.e. those able to carry a steady state flux (args: space-separated list of query reactions; if not provided, all reactions are considered)
  • Plot: iteratively compute the optimal objective function values corresponding to the incrementing/decrementing flux rates of the query reaction (args: `reaction from to step', e.g. `r2 0 1 0.1')
  • Plot 3D: iteratively compute the optimal objective function values corresponding to the incrementing/decrementing flux rates of two query reactions (args: `reaction1 from to step reaction2 from to step', e.g. `r2 0 1 0.1 r3 -1 1 0.2')
  • Minimal substrate sets: compute one or more minimal (i.e. irreducible) sets of external substrates which are sufficient for a given linear combination of external metabolites (supplied as the objective function) to be produced. E.g. if the objective function is biomass, the result would represent minimal media sufficient for growth. Note that some results may correspond to product compositions including additional by-products, not defined in the objective function (e.g. water and CO2). The method uses the MILP solver only. The first argument is the number of sets to be calculated (1 by default); the second argument is a scaling factor (see `Elementary modes'; args: `reaction [number, scale_factor']).
  • Minimal product sets: compute one or more minimal (i.e. irreducible) sets of external products which can be released while a given linear combination of external metabolites (supplied as the objective function) is consumed. E.g. if the objective function is "glucose + 2 water", the result in a fermenting organism would represent minimal sets of fermentation products. Note that all substrates must be defined, since no additional co-substrates are allowed. The arguments are identical to those for minimal substrate sets.
  • Unconserved metabolites: detect the unconserved metabolites; e.g. those whose molecular masses cannot be simultaneously assigned positive values (no args)
  • Orphan metabolites: detect the orphan metabolites (i.e. those used by less than two reactions)
  • Connected components: detect the weakly connected components
  • iMAT method: flux Activity Analysis by shilomi’s method which gives a qualitative prediction about flux activity state for reactions and genes without requiring the biomass synthesis reaction. Apart from genome-scale model, shlomi only needs absolute gene expression data which should reflect the absolute transcript abundances of genes under one particular growth condition. Parameter ‘Threshold’ is positive threshold of active flux which is used to judge if a reaction is active or not. If not set, the default value is 1. (args: space-separated list of query reactions and genes; if not provided, all reactions and genes are considered).
  • Fast iMAT method: an alternative method to iMAT method which is much faster than shlomi’s method but only for reactions, the method relies on MILP. (args: space-separated list of query reactions; if not provided, all reactions are considered).
  • GIMME FBA: the objective function of GIMME method is to minimize the fluxes of lowly expressed reactions weighted by the deviations of reaction expression state from predefined threshold for low expression (default 12). The output is similar as FBA. The objective value is inconsistency score indicating the degree of disagreement between the gene expression data and the assumed objective function under specified required functionalities. (args: space-separated list of query reactions; if not provided, all reactions are considered).
  • GIMME FVA: the FVA version of GIMME method
  • GIM3E: predict reaction flux activities using metabolic constraints, the output is similar as FVA. (args: space-separated list of query reactions; if not provided, all reactions are considered).
  • SGNI: predict Strong Gene-Nutrient Interactions (sGNI) which uncover the dependence of gene essentiality on the presence/absence of nutrients given a growth medium of interest. The algorithm only needs a well-defined genome-scale model (args: biomass reaction; space-separated list of query genes; if not provided, all reactions are considered).
  • WGNI: predict Weak Gene-Nutrient Interactions (sGNI) using sampling approach in which a set of growth media is randomly sampled and normal FBA method are applied to predict gene knockout effects under each sampled medium (args: biomass reaction; space-separated list of query genes; if not provided, all reactions are considered).
  • DPAplot: find producibility plot for Differential Producibility Analysis (DPA). The goal is to identify a set of genes that participate in the production/consumption of each metabolite. This is a binary matrix that links genes with metabolites on the basis of whether or not each gene is essential for production/consumption of each metabolite (no args).
  • DPAsig: from producibility plot produced by dpaplot and by incorporating microarray gene expression data, here we are able to compute a value called ‘metabolite signal’ which reflects the metabolic state. The metabolite signal can be calculated as the median of associated gene expression ratios of its associated up/down-regulated genes (no args).

Analysis of iMAT method, Fast iMAT method, GIMME FBA/FVA and DPAsig needs expression files which can be opened at File > Open expression. Problem DPAsig also needs an output of DPAplot which can be open at File > Open DPAplot. Note that the analysis of iMAT method, Fast iMAT method, SGNI, WGNI, DPAplot and DPAsig can only have one record in problem table. When you choose problem SGNI and WGNI a tab titled ‘Nutrients’ will be automatically popped up, in which you can specify the nutrients of growth media to be tested (as ‘?nutrients’ in problem table) and present/absent nutrients (as ‘$nutrients’ in problem table), then write into problem table.

Using the toolbar   Although the `Problem' table can be edited directly, JyMet provides a toolbar which facilitates the definition of objective functions and argument lists in a problem-specific format. To use the toolbar, select the analysis method in the Analyse menu; the corresponding toolbar will appear. Then enter the objective function and argument values wherever indicated. For Plot 3D, enter pairs of values delimited by spaces, e.g. Reaction: r1 r2; From: -1.0 0; To: 1 2; Step: 0.1 0.2. If the second argument's value is equal to the first one, it may be skipped. You can see SFM for specific problem type and the toolbar are basically self-explanatory.

The `Reduce matrix' checkbox specifies whether the stoichiometry matrix has to be reduced (see SFM for details). The `Comments' checkbox specifies whether comments have to be included into the solution output. Once the toolbar fields are filled, select Solve > Write problem; the problem will be written into the `Problem' table and can be further edited there. You can also clear the problem table by clicking Solve > Clear problem.

Running the solver   Once the problem is defined, select Solve > Solve to launch the solver. While the solver is running, it can be suspended by selecting Solve > Stop. Simultaneous solution of problems is not allowed. You can choose the different solver/algorithms in the `Solver' combo box on the toolbar , wherever they are applicable. For GLPK solver you can choose algorithms: Simplex, Exact or MILP, and Gurobi is for Gurobi optimizer which can automatically choose the simplex or MILP depends on problems (see SFM for details). You can set parameter Dual&Primal feasibility tolerance for Gurobi solver by selecting Slove > Tolerance, tightening this tolerance can produce smaller constraint violations (default: 1e-6, Min: 1e-9, Max: 1e-2). Note that Gurobi solver needs to be pre-installed, it is free for academics. 

Solution output   Once the solver exits, its output is written into a new tab which appears rightmost in the model tab and whose title is the name of the analysis method. The output format is method-specific, but the column names are self-explanatory. Each record have corresponding record sub-tab and there is tab ‘INFO.’ which gives information about analysis type and problem records and corresponding SurreyFBA command line and running time spent. You can save the output table through File > Save table, or you can save the whole SurreyFBA output for current solved problem by saving the temporary output file ‘out.tmp’ in directory ‘/jymet’ as another file name. 

 

4  Visualisation

JyMet can graphically represent models and solution results in network diagrams or charts.

Graph visualisation   This functionality is intended for the visualisation of whole metabolic networks and their subnetworks. Each vertex in a graph corresponds to an individual element: gene, enzyme, reaction or metabolite and is connected with the vertices representing its parents (if present) and children.

To draw a graph, select the cells containing the IDs of the elements to be included. These cells can be selected in any table, including `Problem' tables and those representing results. Note that an element is included with its children (so, a gene is included with the enzymes encoded, the reactions catalysed by these enzymes and the metabolites involved in their equations), but not with its parents. The vertices are named by default with the names of the genes, enzymes or metabolites or the reaction IDs. However, if additional cells are selected, their contents are appended to the names, delimited by semicolons (e.g. to visualise an elementary mode with the flux rates, select the reaction IDs and the flux rates in the solution output table).

The program uses two tools for graph drawing: the built-in JGraph library, and the Graphviz tool, which needs to be pre-installed. To draw a graph using JGraph, select View > Layout and one of the radio buttons specifying the layout methods; The graph drawing algorithm considers some metabolites as `logical', e.g. every occurence of such a metabolite in a reaction is represented by a separate vertex. This reduces the connectedness of the networks and hence improves the readability of the diagrams. The list of logical metabolites can be edited manually in the file `settings.py', which can be found in the installation package. To draw it using Graphviz, select View > Layout > DOT; in both cases, the graph will appear in a new tab. To save it in a file, select File > Save table; the file format will depend on the file extension (e.g. png, bmp or jpg).

Mapping results   Apart from JGraph layouts, you can also choose ‘Custom’ to load the user-built network graph to map the result on it. Currently solution results of FBA, FVA, iMAT method, Fast iMAT method, GIMME FBA, GIMME FVA and GIM3E can be mapped on graph, which are explained as follows:

  • For FBA and GIMME FBA, the fluxes of selected reactions are mapped on reaction nodes and edges between reactions and metabolites indicating the metabolite fluxes which are calculated as its reaction flux multiplied by its coefficient.
  • For FVA, the flux ranges of selected reactions are mapped on reaction nodes, such as 2.89~50.99 meaning from minimal flux 2.89 to maximal flux 50.99.
  • For iMAT method, the ACT_LEVs of selected reactions are mapped on reaction nodes, such as 2f,-3r where ‘2f’ is ACT_LEV of 2 in forward direction and ‘-3r’ is ACT_LEV of -3 in reverse direction if this is reversible reaction.
  • For Fast iMAT method and GIMME FVA, the Activities of selected reactions are mapped on reaction nodes, where ‘f’ and ‘r’ are for active in forward and reverse direction, ‘-1’ is for inactive and ‘0’ is for undetermined activity.

You can manipulate your graph by right clicking the mouse where you can zoom in/out your graph, this can also be achieved by double clicking your mouse (right for zoom out and left for zoom in); you can select and search nodes or edges by all vertices/edges or by name; you also can highlight the edges for FBA and GIMME FBA by set a set of flux thresholds to classify the edge thickness, such as “1 10 50’ (separated by one space) which gives 4 levels of edges thickness. 

To draw it using Graphviz, select View > Layout > DOT; in both cases, the graph will appear in a new tab. To save it in a file, select File > Save table; the file format will depend on the file extension (e.g. png, bmp or jpg).

The graph drawing algorithm considers some metabolites as `logical', e.g. every occurence of such a metabolite in a reaction is represented by a separate vertex. This reduces the connectedness of the networks and hence improves the readability of the diagrams. The list of logical metabolites can be edited manually in the file `settings.py', which can be found in the installation package.

Building graph   To build a new network graph from scratch for a model or part of a model, currently you can only use one of JGraph layouts to open a graph in JyMet then adjust the graph by following operations:

  • Drag the nodes or edges to desired positions.
  • Press Control key at the same time drag the nodes or edges to copy them.
  • Move mouse over the nodes or edges then right click to select ‘Delete’ to delete them.
  • Move mouse to a start node, keep pressing left button of mouse and move it to an end node to get an edge between 2 nodes.
  • Double click the text in nodes/edges to edit them.

After that, you can save your graph as a text file by selecting File>Save graph. This file describing the nodes and edges information related to the graph. You can also modify your graph by editing its graph text file. The format of graph file is as follows. The first line gives graph name, and then each line describes a node/edge by the tab-delimited columns. For nodes the meaning of 8 columns from left to right are as follows:

  • With ‘Node’ indicating a node.
  • Node id, a metabolite can have many copies with different ids related to different reactions.
  • Node name presented in the graph.
  • Node position in X axis, the distance to the left side of graph.
  • Node position in Y axis, the distance to the up side of graph.
  • Height of node outline.
  • Width of node outline.
  • A set of features describing appearance of nodes which are self-explained.

For edges the meaning of 6 columns from left to right are as follows:

  • With ‘Edge’ indicating an edge.
  • Edge id, normally a number produced by JyMet.
  • Edge information between 2 nodes such as stoichiometry, flux. 
  • Node id the edge starts from.
  • Node id the edge points to.
  • A set of features describing appearance of edges which are self-explained.

Edge types: the normal edge is solid line with triangle arrow; the edge from an activator is dashed line with triangle arrow; the edge from an inhibitor is dashed line with oval arrow.

 

Charts   One or more solution outputs can be visualised in a chart, whose type depends on the analysis method:

  • FBA results, iMAT method, WGNI, DPAplot, DPAsig, elementary modes, minimal substrate and product sets are shown in bar charts.
  • FVA and Fast iMAT method results are shown in box and whisker charts.
  • Plot results are shown in line plots.
  • Plot 3D results are shown in grayscale block plots.

The first three types of the charts listed above can cover multiple results from different output tables (e.g. in a line plot, they are shown as different curves). The chart legends are extracted from the output comments, if they are selected.

To visualise one or more results in a chart, select the corresponding rows in the output tables and select View > Plot; the chart will appear in a new tab. For DPAplot and DPAsig output tables you can select specific row and column to plot. To edit the chart, click the right mouse button in it and select Properties in the popup menu. To save the chart in a png file, select either File > Save table, or Save in the context menu.

 

5  QSSPN

The menu QSSPN contains the operations implemented for QSSPN (Quasi-Steady State Petri Nets) software (see website at http://sysbio3.fhms.surrey.ac.uk/qsspn/index.html). To run the QSSPN, firstly you need to open a GSMN model of SurreyFBA reaction table file (File > Open model) which will be used by QSSPN model, then open corresponding QSSPN model (QSSPN  > Open QSSPN) or import a SPEPT model file built by Snoopy software (QSSPN  > Import SPEPT), then you have to load a QSSPN control file (QSSPN  > Load control). To run QSSPN software you just click QSSPN  > Run and stop it by QSSPN  > Stop.

You can edit QSSPN model file and control file by modifying the field values and also you can insert or delete rows through Edit > Insert rows and Edit > Delete rows.  After editing you can save the QSSPN model and Control parameters into files in QSSPN formats. When you get output of QSSPN, you can use QSSPN  > Plot trajectory to draw a plot showing multiple trajectories by selecting the target node columns and time span. You can even open a saved output file (QSSPN > Open output) to view and plot them.

 

6  Final remarks

System requirements   JyMet is written in Jython (a nice hybrid of Python and Java), and is intended to be platform-independent, only requiring a Java virtual machine to be installed. So far it has been successfully tested on various versions of Linux, Windows and Mac. Two versions of software are provided, one is called sfba-glpk which only uses GLPK library, and the other is called sfba-grb which uses GLPK and Gurobi libraries, note that before use this version you have to install licensed Gurobi solver.
